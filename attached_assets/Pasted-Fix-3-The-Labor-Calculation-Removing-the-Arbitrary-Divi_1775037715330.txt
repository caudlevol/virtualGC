Fix 3 — The Labor Calculation (Removing the Arbitrary Division)
What is actually broken:
In costLookup.ts, labor is calculated as:
TypeScript
const laborCostPerUnit = (blsLaborRate * productivityFactor) / 4;
Dividing by 4 assumes every unit of work takes exactly 15 minutes. Installing a toilet takes 2 hours. Painting one square foot takes 3 minutes. These are wildly different, and any real GC or experienced agent will immediately know the numbers are wrong.
The fix — add real production rates:
Replit Prompt 3 — Fix the Labor Cost Calculation Logic
Step 1 — Schema (lib/db/src/schema/costEngine.ts):
Add a new column to the material_costs table:
TypeScript
laborHoursPerUnit: real("labor_hours_per_unit").default(1.0)
Run the Drizzle migration after adding this column.
